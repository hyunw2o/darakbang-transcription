# 출시 전략: 다글로(Daglo) 대비 차별화 + 지식재산권/법적 리스크 점검

기준일: 2026-02-13

## 1) 핵심 결론 (요약)
- 현재 서비스 방향(설교/교회 특화 STT)은 다글로의 범용 업무 자동화 포지션과 다르므로, **포지셔닝 자체는 차별화 가능**합니다.
- 다만 출시 시점의 실질 리스크는 기능 유사성보다 **브랜드/카피/UI 모사, 데이터 권리(녹음 동의), 개인정보 처리 고지 미흡**에서 발생할 가능성이 큽니다.
- 따라서 초기 30일은 기능 확장보다 **법적/운영 안전장치 + 특화 기능 고도화**를 우선해야 합니다.

## 2) 다글로 대비 차별화 포지셔닝

### 권장 포지셔닝 문장
- "범용 회의록 툴"이 아니라 **"교회 설교/훈련 콘텐츠 특화 기록·교정·주보 생성 도구"**로 정의합니다.

### 차별화 축 (실행 우선순위)
1. 도메인 특화 정확도
- 다락방/신학 용어 사전, 성경 구절 포맷 정규화, 설교체 문장 교정.
- 지표: 도메인 단어 오류율(TER) 별도 측정.

2. 산출물 특화
- 단순 전사본이 아니라 주보용 요약, 핵심 메시지/기도제목/적용포인트 포맷 자동화.
- 지표: 편집 시간 절감(분/건).

3. 운영 방식 특화
- 교회별 사전(커스텀 용어집), 교역자 프로필별 교정 스타일.
- 지표: 재수정률 감소.

4. 데이터 신뢰
- 업로드 파일 보관기간 설정, 즉시 삭제 옵션, 열람 권한 최소화.
- 지표: 보관 정책 준수율 100%.

## 3) 지식재산권/법적 리스크 점검 (출시 전 필수)

주의: 아래는 실무 점검용이며 법률 자문이 아닙니다. 출시 전 변리사/변호사 검토 권장.

### A. 상표/브랜드 리스크
- [ ] 서비스명/로고/슬로건이 기존 서비스와 혼동되지 않는지 점검.
- [ ] KIPRIS에서 유사 상표(한글/영문 발음 포함) 선검색.
- [ ] 출시명 확정 즉시 상표 출원 검토(최소 핵심 클래스).

실무 기준
- "다글로", "Daglo" 유사 표기/발음 연상 명칭은 피함.
- 마케팅 카피도 "대한민국 1등" 같은 경쟁사 대표 문구 모사는 피함.

### B. UI/카피/콘텐츠 저작권 리스크
- [ ] 경쟁사 랜딩 카피/아이콘/레이아웃을 직접 복제하지 않았는지 점검.
- [ ] 기능 설명 문구를 자체 작성(번역 복붙 금지).
- [ ] 데모 이미지/샘플 음성의 권리 보유 여부 확인.

실무 기준
- "아이디어/기능" 유사 자체보다, 구체 표현(문구/디자인/리소스) 모사가 분쟁 포인트가 됩니다.

### C. 음성 데이터/개인정보 리스크
- [ ] 업로드 시 녹음 당사자 고지 및 동의 책임 안내 문구 제공.
- [ ] 개인정보처리방침에 수집항목/목적/보관기간/위탁·국외이전 반영.
- [ ] 관리자 접근 로그, 삭제 요청 처리 프로세스 준비.

실무 기준
- 음성 파일에는 개인정보가 포함될 수 있으므로 최소수집·목적내 이용·보관기간 통제가 필요합니다.

### D. 제3자 모델/API 약관 리스크
- [ ] Gemini/OpenAI/Supabase 이용약관 기준으로 금지행위·데이터 처리조건 확인.
- [ ] 학습 사용 여부(옵트아웃 포함), 로그 보관 정책을 약관과 맞춤.
- [ ] 고객 안내문에 "외부 AI 처리" 고지 여부 반영.

## 4) 출시 전 보완 우선순위 (P0/P1/P2)

### P0 (출시 전 반드시)
- [ ] 서비스명/로고 확정 + 상표 선검색
- [ ] 개인정보처리방침/이용약관/운영정책 공개
- [ ] 업로드 동의 체크박스(본인 책임 및 제3자 고지)
- [ ] 파일 보관기간 기본값 설정(예: 30일) + 즉시 삭제 API
- [ ] 관리자 화면 접근통제 및 로그

### P1 (출시 1개월 내)
- [ ] 교회별 커스텀 용어사전 UI
- [ ] 주보 템플릿(요약 포맷 3종)
- [ ] 정확도 리포트(원문/교정문 diff)
- [ ] 사용자별 작업 이력 검색/필터

### P2 (확장)
- [ ] 화자 분리 고도화
- [ ] 다국어 자막 패키지
- [ ] 팀/교회 단위 권한관리
